[7. AI 활용 코딩 가이드 (권장)]

AI를 활용한 코딩은 허용하되, 아래 가이드를 준수한다.

[1) 프롬프트 작성 팁]

요구사항을 최대한 구체적으로 작성한다.
(페이지명, 기능, 사용 기술, 조건 등을 명확히 기재)

한 번에 많은 기능을 요청하기보다,
작은 단위로 나누어 요청 후 단계적으로 확장한다.

기존 코드가 있을 경우 반드시 함께 제공한다.
(전체 코드 또는 수정이 필요한 파일 단위)

결과물은 바로 사용하지 말고,
코드 구조와 동작을 반드시 이해한 후 적용한다.

[2) 유지보수가 쉬운 코드 작성 가이드]

코드의 목적이 드러나도록 변수명과 함수명을 작성한다.
(의미 없는 약어, 축약어 사용 지양)

하나의 함수는 하나의 역할만 수행하도록 작성한다.

하드코딩된 값은 상수로 분리한다.
(색상, 크기, 문자열 등)

반복되는 코드는 함수 또는 공통 컴포넌트로 분리한다.

불필요하게 복잡한 로직은 지양하고,
읽기 쉬운 구조를 우선한다.

[3) AI 코드 검증 체크리스트]

AI가 생성한 코드를 사용할 경우 아래 사항을 반드시 확인한다.

프로젝트 기술 스택과 일치하는지 확인

사용하지 않는 코드 또는 불필요한 라이브러리 포함 여부 확인

기존 코드 스타일 및 네이밍 규칙과 일치하는지 확인

콘솔 에러 및 경고 메시지 발생 여부 확인

[4) 협업 시 주의사항]

AI가 생성한 코드라도 책임은 작성자 본인에게 있음

이해하지 못한 코드는 merge 금지

수정 범위가 큰 경우, 커밋 메시지 또는 문서에 AI 사용 여부 명시 권장

[5) 권장 마인드셋]

AI는 '대체 수단'이 아닌 '보조 도구'로 사용한다.

빠른 구현보다 유지보수와 가독성을 우선한다.

팀원이 코드를 처음 보더라도 이해할 수 있는 코드를 목표로 한다.
