📁 정적 이미지 리소스 디렉토리 규칙
1. 목적

프로젝트에서 사용되는 모든 정적 이미지 파일을 관리하기 위한 디렉토리

2. 디렉토리 구조

페이지 단위 또는 컴포넌트 단위로 디렉토리를 생성한다.

디렉토리 이름은 이미지가 사용되는 위치를 명확히 알 수 있도록 작성한다.

/images
 ├─ MainPage
 ├─ LoginPage
 ├─ Header
 └─ Footer

3. 파일 명 규칙

파일명은 이미지의 역할이나 위치가 바로 이해되도록 직관적으로 작성한다.

의미 없는 번호 사용은 지양한다.
(❌ img1.png, ⭕ hero-banner.png)

4. 네이밍 컨벤션

영문 사용

kebab-case(소문자-하이픈) 권장

공백, 특수문자 사용 금지

5. 예시
MainPage/hero-banner.png
MainPage/section-feature.png
Header/logo.png
LoginPage/bg-image.png
